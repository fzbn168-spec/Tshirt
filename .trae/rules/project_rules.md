# 项目重构规则（SOP 版）

本项目正在进行外贸独立站重构，目标实现 **零售 + 小额批发 + 大额 B2B 混合模式**。请严格遵循以下规则。

## 1. 项目背景
- 技术栈：Next.js (App Router) + NestJS + Prisma + PostgreSQL + Tailwind CSS。
- 现有基础：已有 B2B 询价（RFQ）功能，现需扩展零售购物车、直接结账、阶梯价、多国尺码表等。
- 分支策略：所有重构在 `refactor` 分支进行，`main` 为稳定备份。

## 2. 开发流程（按阶段执行）
必须遵循 **P0 → P1 → P2 → P3** 顺序，不可跳步。每个阶段完成后方可进入下一阶段。

### P0 阶段（核心交易）
- 数据库：添加购物车、零售订单、发货记录、标签、商品多图等模型。
- 后端 API：购物车 CRUD、零售订单创建/查询、发货记录管理、标签 CRUD。
- 前端商城：产品页加购按钮、购物车页、结账页、订单成功页、用户中心订单列表。
- 管理后台：零售订单管理、商品价格扩展（零售价/批发价）、标签管理。
- 支付：集成 PayPal 沙箱。

### P1 阶段（增强转化）
- 粘性购买栏
- 热卖标签展示
- 物流跟踪（分批发货）
- 客户定制需求（图片/文字上传）
- 库存管理（前台显示库存状态）

### P2 阶段（增长引擎）
- 数据看板（销售额、转化率等）
- 客户分群（RFM 分析）
- 邮件营销（SendGrid 集成）

### P3 阶段（合规与效率）
- 操作日志
- 发票系统（PDF）
- 批量操作商品
- 安全加固

## 3. 代码规范
- **注释**：所有新增函数、组件、API 必须添加**中文注释**，说明作用、参数、返回值。
- **命名**：使用有意义的英文命名（驼峰），禁止拼音或缩写。
- **类型**：使用 TypeScript 严格模式，禁止 `any`。后端 DTO 必须使用 `class-validator` 校验。
- **目录结构**：遵循现有模块化结构，新功能放在对应模块下。
- **响应式设计**：所有前端界面必须采用响应式设计，优先移动端（Mobile First）。使用 Tailwind CSS 的响应式类（如 `sm:`、`md:`、`lg:`）实现布局适配。关键功能（如购物车、结账、粘性购买栏）需在移动端有优化的交互版本（例如，筛选侧边栏折叠、表格转卡片）。
