基于30年行业经验，针对**“真实支付配置”**和**“种子用户内测”**这两个关键环节，我为您制定了以下实战方案。

### 1. Stripe 密钥配置 (The "Money" Part)

要让您的平台真正收钱，必须从“沙盒模式”切换到“生产模式”。

*   **配置清单**:
    *   **后端**: 需要 `STRIPE_SECRET_KEY` (以 `sk_live_` 开头) 来创建支付意向。
    *   **前端**: 需要 `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` (以 `pk_live_` 开头) 来渲染信用卡组件。
*   **安全策略**:
    *   **绝对禁止**将 Key 直接写在代码里。
    *   必须通过环境变量 (`.env`) 注入。
    *   对于 Docker 环境，我将为您创建一个 `setup-production-env.sh` 脚本，安全地将密钥注入容器。

### 2. 种子用户内测策略 (The "People" Part)

B2B 鞋服外贸不同于 C 端应用，不需要大规模撒网，而是需要**“白手套式” (White-Glove)** 的精准邀请。

*   **邀请策略**: **封闭式内测 (Closed Beta)**。不开放公开注册，由管理员手动创建账号并发送邀请函。
*   **工具支持**:
    *   为了避免您手动操作数据库，我将编写一个 **“客户开户脚本” (`onboard-beta-client.ts`)**。
    *   **功能**: 一键生成 公司 + 管理员账号 + 随机密码，并输出一段“邀请话术”，您可以直接复制发给客户。

### 📅 执行计划 (Execution Plan)

#### 任务 A: 生产环境配置脚本
*   创建 `scripts/setup-production-env.sh`。
*   功能：交互式询问 Stripe Key，并将其安全地写入 `.env` 文件或 Docker 环境变量配置中。

#### 任务 B: 内测客户开户脚本
*   创建 `backend/scripts/onboard-beta-client.ts`。
*   功能：接受 `公司名` 和 `邮箱` 参数，自动创建数据，并生成“欢迎邮件”模版。
